Here's the breakdown — the arithmetic in that line is actually correct; the issue is what numbers were plugged in, not the method.

Checking the arithmetic itself:

$500,000 − $314,000 = $186,000 ✓
$186,000 × 0.078 = $14,508 ✓
$29,754 − $14,508 = $15,246 ✓
So mechanically, nothing is wrong with the formula or the maths. The taper rate (0.078 = 7.8% annually, equivalent to $3/fortnight per $1,000) is also the correct rate to use — that part isn't an error either.

What's actually wrong: the two input figures are outdated, not the formula.

$29,754 (full pension) — this was the annual full Age Pension rate for an earlier period, but it's since increased. As of the March 2026 indexation, the current full single Age Pension is $31,223/year.
$314,000 (asset threshold) — this was the correct full-pension asset threshold for a single homeowner for an earlier period, but it's also since increased. As of March 2026, it's $321,500.
Both numbers moved because Centrelink indexes Age Pension rates and asset thresholds twice a year (March and September) in line with the cost of living — so a calculation that was accurate six or twelve months ago will look "wrong" today purely because the inputs have aged, not because the formula was flawed.

So the honest verdict on that line:

Formula/structure: correct
Taper rate 0.078: correct
Arithmetic: correct
The two dollar figures plugged in: stale — from an earlier indexation period, not current
If you rerun it with today's figures ($31,223 full pension, $321,500 threshold), you get roughly $17,301/year instead of $15,246 — the gap is entirely due to using older numbers, not a flaw in the calculation method.
